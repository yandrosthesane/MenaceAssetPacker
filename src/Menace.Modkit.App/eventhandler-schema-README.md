# EventHandler Schema - Quick Start Guide

## Overview

This directory contains a comprehensive schema for all EventHandler types found in Menace's game data. The schema was auto-generated by analyzing 1,289 EventHandler instances across 710 templates (121 perks + 589 skills).

## Files

- **`eventhandler-schema.json`** - The main schema file (66KB, 3,183 lines)
- **`eventhandler-schema-report.md`** - Detailed analysis report with statistics and findings
- **`eventhandler-schema-README.md`** - This file

### Programmatic Usage (Python)

```python
import json

with open('eventhandler-schema.json', 'r') as f:
    schema = json.load(f)

# Get all types
types = schema['eventHandlerTypes'].keys()

# Get info for a specific type
attack_type = schema['eventHandlerTypes']['Attack']
print(f"Attack has {len(attack_type['fields'])} fields")
print(f"Found in {attack_type['instanceCount']} instances")

# Get field info
damage_field = attack_type['fields']['Damage']
print(f"Damage field type: {damage_field['type']}")
print(f"Sample value: {damage_field['sample']}")
```

### Programmatic Usage (C#)

```csharp
using System.Text.Json;

var json = File.ReadAllText("eventhandler-schema.json");
var schema = JsonDocument.Parse(json);

var types = schema.RootElement
    .GetProperty("eventHandlerTypes")
    .EnumerateObject();

foreach (var type in types)
{
    Console.WriteLine($"{type.Name}: {type.Value.GetProperty("instanceCount")}");
}
```

## Schema Structure

```json
{
  "eventHandlerTypes": {
    "TypeName": {
      "instanceCount": 123,
      "fields": {
        "FieldName": {
          "type": "string|int|float|enum|array|object|null",
          "nullable": true,              // Optional
          "values": [0, 1, 2],           // For enums
          "elementType": "string",       // For arrays
          "commonValues": ["a", "b"],    // For strings
          "sample": <value>              // Example value
        }
      }
    }
  }
}
```

## Common Patterns

### Boolean-like Enums

Many fields use `[0, 1]` enum values representing false/true:
- `AddSkill.ShowHUDText`
- `Attack.ApplyMode`
- `DisableSkills.HideDisabledSkills`

### Event Timing

Multiple types have `Event` fields with enum values representing when to trigger:
- `AddSkill.Event: [0, 1, 2, 3, 4, 5, 6]`
- `DisplayText.Event: [0, 1, 2, 3]`
- `SpawnTileEffect.Event: [0, 1, 2]`

### Multipliers

Fields ending in `Mult` typically have `1.0` as default:
- `Attack.AccuracyMult: [0.8, 0.85, 0.9, 1.0]`
- `Attack.DamageMult: [0.7, 1.0, 1.2]`

### Dropoff Values

Negative values indicate range-based degradation:
- `Attack.AccuracyDropoff: [-12.0, ..., 0.0, 1.0]`
- `Attack.DamageDropoff: [-1.5, ..., 0.0, 0.8]`

### Tag Arrays

Many types support tag filtering:
- `TargetRequiresOneOfTheseTags: string[]`
- `TargetCannotHaveOneOfTheseTags: string[]`
- `RequiredTags: string[]`
- `ForbiddenTags: string[]`

## Field Types Explained

| Type | Description | Example |
|------|-------------|---------|
| `string` | Text value | `"effect.berserk"` |
| `int` | Whole number | `42` |
| `float` | Decimal number | `3.14` |
| `bool` | True/false (rare) | `true` |
| `enum` | Limited set of values | `[0, 1, 2]` |
| `array` | List of values | `["tag1", "tag2"]` |
| `object` | Complex structure | `{"x": 1, "y": 2}` |
| `null` | Always null | `null` |

### Type Modifiers

- **`nullable: true`** - Field can be null or the specified type
- **`values: [...]`** - For enums, lists all possible values
- **`elementType: "type"`** - For arrays, the type of elements
- **`commonValues: [...]`** - For strings, frequently seen values

## Important Notes

### Enum vs. Bitmask

Some "enum" fields may actually be bitmasks:
- **Attack.ElementsHit:** `[0, 1, 2, 3, 9]` - Note the jump to 9
- **SetEntityFlag.Flag:** `[4, 128, 240, 512, ...]` - Powers of 2

These should be treated as flags, not simple enums.

### Rare Types

63 types have fewer than 3 instances in the game data:
- May represent special mechanics
- Could be cut/unused features
- Might be DLC-specific
- Test thoroughly when using

See `eventhandler-schema-report.md` for complete list.

### Object Fields

Object fields need manual investigation for internal structure:
- **Vector3**: `{x: float, y: float, z: float}`
- **AnimationCurve**: Unity animation curve data
- **SoundReference**: Audio asset reference

The schema identifies these but doesn't provide sub-structure.

### Null Fields

4 fields are always null in observed data:
- May be deprecated/unused
- Could be set by game code
- Might be DLC placeholders
